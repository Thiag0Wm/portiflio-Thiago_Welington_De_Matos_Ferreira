✈️ Prompt Battle - Engenharia de Prompt (XML)
📝 Descrição do Projeto
Este projeto foi desenvolvido como parte da disciplina de Engenharia de Prompt e Aplicações em IA. O objetivo principal foi realizar um estudo comparativo entre diversos modelos de linguagem (LLMs) utilizando uma estrutura de Prompt em XML para a criação de uma Single Page Application (SPA) de pacotes de viagens.

O experimento foca na capacidade das IAs em interpretar diretrizes técnicas rigorosas, como o uso de paletas de cores específicas, tipografia híbrida e componentes funcionais complexos (carrinhos de compras e sistemas de pagamento), tudo gerado a partir de uma única instrução estruturada.

🚀 Diferenciais e Funcionalidades do Prompt
Arquitetura XML: Utilização de tags estruturadas (<tarefa>, <objetivo>, <diretrizes_design>) para garantir máxima aderência da IA às regras de negócio.
Design System Integrado: Definição de paleta em Roxo e Laranja, com layout responsivo e profissional.
Componentes Complexos: Inclusão de lógica para calendário de datas, seleção de aeroportos, tela de login e fluxo de checkout.
Análise Comparativa: Metodologia de teste aplicada em 7 modelos diferentes para validar a precisão do HTML e a criatividade do conteúdo.
🛠️ Modelos de IA Avaliados
O prompt estruturado foi testado nas seguintes ferramentas para medir eficiência e bugs:

Claude: Eleito o melhor modelo pela compreensão superior da estrutura e criatividade profissional.
Grok: 2º melhor resultado, com alta fidelidade visual e inclusão de elementos gráficos.
DeepSeek: Boa precisão técnica, porém com baixa criatividade visual.
Outros: GPT, Gemini, Qwen e Maritaca.
📊 Resultados e Aprendizados
O projeto revelou dados cruciais sobre a "personalidade" técnica de cada IA e a eficiência no consumo de recursos.

Verbosidade: Identificou-se uma variação discrepante no uso de tokens, indo de 1.200 (Maritaca) até 9.200 (Claude) para a mesma tarefa.
Acurácia Técnica: O modelo Claude foi o único que não apresentou bugs de sintaxe ou funcionalidades quebradas no carrinho de compras.
Conclusão de Prototipagem: O estudo definiu o Claude como a ferramenta preferencial para geração de códigos complexos e prototipagem rápida de interfaces.
🔧 Como Replicar o Experimento
Utilize o código de prompt estruturado presente no arquivo XML deste repositório.
Insira o conteúdo em um modelo de linguagem de sua escolha (recomenda-se o Claude para fidelidade total ao design).
O resultado será um arquivo HTML único com CSS interno pronto para visualização no navegador.
